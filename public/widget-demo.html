üíª Integration Methods

Method 1: Simple Auto-Init

Just add a div and the script - the widget auto-initializes:

<!-- Add this div where you want the widget --> <div id="xertiq-verify-widget"></div> <!-- Add the widget script --> <script src="https://your-domain.com/xertiq-widget.js"></script>

Method 2: Auto-Verify with Document ID

Use data attributes to pre-fill the document ID:

<!-- Widget with auto-verification --> <div id="xertiq-verify-widget" data-doc="cert_certificate_Name_123_abc" ></div> <script src="https://your-domain.com/xertiq-widget.js"></script>

Method 3: Programmatic Initialization

Full control over widget initialization:

<div id="my-verification-widget"></div> <script src="https://your-domain.com/xertiq-widget.js"></script> <script> // Initialize after script loads XertiQWidget.init({ container: 'my-verification-widget', docId: 'cert_certificate_Name_123_abc', // Optional apiUrl: 'https://your-api-domain.com/api' // Optional }); </script>

Method 4: React Component

Integrate with React apps:

import React, { useEffect, useRef } from 'react'; const XertiQVerifyWidget = ({ documentId, apiUrl }) => { const containerRef = useRef(null); useEffect(() => { // Load widget script if not already loaded if (!window.XertiQWidget) { const script = document.createElement('script'); script.src = 'https://your-domain.com/xertiq-widget.js'; script.async = true; script.onload = () => initWidget(); document.body.appendChild(script); } else { initWidget(); } function initWidget() { if (containerRef.current && window.XertiQWidget) { window.XertiQWidget.init({ container: containerRef.current, docId: documentId, apiUrl: apiUrl }); } } }, [documentId, apiUrl]); return <div ref={containerRef} />; }; export default XertiQVerifyWidget; // Usage: // <XertiQVerifyWidget documentId="cert_certificate_Name_123_abc" />

üìä Why No iFrames?

Advantages over iFrame approach:

  • Better Performance: No additional page load, faster rendering
  • Seamless Integration: Feels like part of your website
  • Easier Styling: Can inherit parent styles or be fully customized
  • Better UX: No scrolling issues or iframe sizing problems
  • SEO Friendly: Content is part of the main page
  • Communication: Direct access to page context (no postMessage needed)
  • Mobile Optimized: Better responsive behavior
